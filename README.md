# Research Log

논문, 기술 자료, 연구 아이디어, 논문화 진행 과정을 시간순으로 정리하는 저장소입니다.

이 저장소의 목적은 완성된 프로젝트 결과물을 보여주는 것이 아니라, 어떤 질문에서 출발했고 어떤 자료를 읽었으며 어떤 방향으로 연구를 발전시켰는지를 남기는 것입니다.

## Role In My GitHub

| Repository | Role |
| --- | --- |
| [`ai-engineering-study`](https://github.com/minbro-01/ai-engineering-study) | 수업 기반 개념 정리와 작은 구현 연습 |
| [`academic-projects`](https://github.com/minbro-01/academic-projects) | 프로젝트 결과물, 코드, 실험 결과를 정리할 전용 저장소 |
| [`research-log`](https://github.com/minbro-01/research-log) | 논문 리뷰, 연구 질문, 논문화 진행, 회고 기록 |

## What Goes Here

| Type | Description |
| --- | --- |
| Paper Reviews | 논문을 읽고 문제의식, 방법, 한계, 적용 가능성을 정리합니다. |
| Literature Notes | 책, 강의, 기술 문서, 정책 자료를 읽으며 연구에 필요한 내용을 정리합니다. |
| Research Ideas | 프로젝트나 수업에서 생긴 질문을 연구 가능한 형태로 다듬습니다. |
| Paper Progress | 논문화 과정에서 주제, 관련 연구, 방법, 실험, 수정 내용을 기록합니다. |
| Weekly Logs | 일정 기간 동안 읽은 자료, 시도한 일, 다음 행동을 남깁니다. |
| Retrospectives | 배운 점과 개선할 점을 정리합니다. |

## What Does Not Go Here

- 프로젝트 전체 코드
- 데이터셋 원본
- 최종 실험 결과 중심의 프로젝트 문서
- 수업자료나 외부 자료의 단순 복사본

위 내용은 필요에 따라 [`academic-projects`](https://github.com/minbro-01/academic-projects) 또는 [`ai-engineering-study`](https://github.com/minbro-01/ai-engineering-study)에 분리합니다.

## Source Policy

- 참고한 자료는 출처를 남깁니다.
- 내가 이해한 내용, 직접 구현한 내용, 프로젝트에서 맡은 역할을 구분합니다.
- 외부 자료를 그대로 복사하지 않고, 내 언어로 다시 정리합니다.
- 아직 확실하지 않은 내용은 가설이나 질문으로 표시합니다.
